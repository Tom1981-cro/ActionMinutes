## Replit Agent Prompt ‚Äî Implement Theme Switcher (Aurora default)

You are working in a **Vite + React + TypeScript** app using **shadcn/ui** and **Tailwind CSS** (Vibrant Enterprise design system). Implement a **Theme switcher** inside **Settings** that lets users choose **one of 5 themes** and also choose **Light/Dark mode**. Make **Aurora** the default theme.

### Goals

1. Add **Theme** selection (5 templates):

   * `aurora` (default)
   * `paper`
   * `grid`
   * `clay`
   * `terminal`
2. Add **Mode** selection: `light` / `dark`
3. Apply theme + mode globally (entire app), not just preview.
4. Persist selections in `localStorage` so they survive refresh.
5. Respect system preference **only on first run** if no saved preference exists (optional but recommended).
6. Add a **Theme preview block** in Settings (small preview of card/button colors).
7. Ensure theme switching updates shadcn tokens (CSS variables) without breaking existing styles.

---

## Constraints / Architecture Rules

* Use **CSS variables** (`--background`, `--foreground`, `--card`, `--primary`, etc.) as shadcn expects.
* Apply theme using attributes on the root element:

  * `<html data-theme="aurora" class="dark">` or `<html data-theme="grid">`
* Do **not** duplicate app styling per page. One global theme system.
* Keep the implementation small and clean: a provider + hook + Settings UI.

### 2) Add Global Theme CSS Tokens

Create:

* `src/theme/themes.css`

Put **all five themes** as CSS variable blocks using:

```css
:root { /* fallback token values */ }
html[data-theme="aurora"] { ...tokens... }
html.dark[data-theme="aurora"] { ...tokens... }
...repeat for each theme...
```

Use the exact token sets from the Theme Gallery demo (Aurora/Paper/Grid/Clay/Terminal).
Important: This file must define all shadcn variables used in your project:

* `--background`, `--foreground`, `--card`, `--card-foreground`, `--popover`, `--popover-foreground`,
* `--primary`, `--primary-foreground`,
* `--secondary`, `--secondary-foreground`,
* `--muted`, `--muted-foreground`,
* `--accent`, `--accent-foreground`,
* `--destructive`, `--destructive-foreground`,
* `--border`, `--input`, `--ring`,
* optionally `--radius`

Also include optional background styling:

* Aurora: subtle radial gradients in light mode, minimal in dark
* Terminal: subtle neon radial gradients in dark mode only

Then ensure `src/main.tsx` imports it:

```ts

```

### 3) Create Theme Provider + Hook

Create:

* `src/theme/theme-store.ts`
* `src/theme/useTheme.ts`
* `src/theme/ThemeProvider.tsx`

Requirements:

* Store shape:

```ts
type ThemeState = { theme: ThemeId; mode: Mode };
```

* Default:

  * `theme = "aurora"`
  * `mode = "dark"` OR detect from system (preferable):

    * if no saved preference, set mode based on `window.matchMedia('(prefers-color-scheme: dark)')`
* Persist keys in localStorage:

  * `am.theme` and `am.mode`
* Apply to `<html>` every time theme/mode changes:

  * `document.documentElement.dataset.theme = theme`
  * toggle `document.documentElement.classList.toggle("dark", mode === "dark")`

Provide functions:

* `setTheme(themeId)`
* `setMode(mode)`
* `resetTheme()` (sets aurora + system mode OR aurora + dark; choose one and document it)

Expose hook:

```ts
const { theme, mode, setTheme, setMode, resetTheme } = useTheme();
```

### 4) Wire Provider Into App Root

In `src/main.tsx` wrap the app:

```tsx
<ThemeProvider>
  <App />
</ThemeProvider>
```

Ensure it applies theme immediately on load to avoid ‚Äúflash‚Äù:

* In ThemeProvider, apply theme in a `useLayoutEffect` on mount.
