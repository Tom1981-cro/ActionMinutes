Add an automated testing system to ActionMinutes that works without GitHub: unit tests, API integration tests, E2E smoke tests, and a deploy “test gate” so deployments fail if tests fail.

STACK
- Unit tests: Vitest
- API tests: Vitest + Supertest (or equivalent)
- E2E: Playwright (headless)
- Add deterministic Mock AI mode used by tests.

REQUIREMENTS
1) TEST STRUCTURE
- /tests/unit:
  - confidence mapping -> status (needs_review/open)
  - reminder rebucketing logic (Today/Tomorrow/Next Week/Next Month/Sometime)
  - RBAC permission function tests
- /tests/api:
  - auth login works
  - create meeting draft
  - extract endpoint with Mock AI enabled returns valid JSON
  - generate drafts endpoint returns expected drafts
- /tests/e2e (Playwright):
  - smoke: login -> create meeting -> extract (mock) -> generate drafts -> open draft -> copy -> toast appears
  - smoke: personal reminders board create + move bucket persists
  - smoke: theme persists after reload

2) DETERMINISM
- Add env flags for tests:
  - AI_FEATURE_ENABLED=false or AI_MODE=mock
  - INTEGRATIONS_FEATURE_ENABLED=false
  - FIXED_CLOCK=true (freeze time in tests)
- Ensure seeded test user + deterministic seeded data for tests.

3) TEST DATABASE
- Use a separate DATABASE_URL_TEST.
- Prefer SQLite for test runs (fast, deterministic), even if production is Postgres.
- Run prisma migrations against the test DB automatically before tests.
- Provide a seed script for test DB.

4) SCRIPTS (package.json)
- "test": runs unit + api
- "test:unit"
- "test:api"
- "test:e2e"
- "test:all": unit + api + e2e
- "db:test:reset": resets test DB + migrations + seed

5) DEPLOY GATE
- Ensure the Replit deploy build/start process runs "npm run test:all" BEFORE creating a release.
- If tests fail, deployment must fail.

6) DOCUMENTATION
- Add a /help/testing page or README section:
  - how to run tests locally in Replit
  - how to set DATABASE_URL_TEST
  - how to run Playwright in headless mode
  - how to use Mock AI mode for tests

7) IMPORTANT
- Do NOT require real Gmail/Outlook OAuth for any tests.
- Do NOT require real AI calls for any tests.
- Use data-testid attributes on key UI elements to make E2E stable.

DELIVERABLES
- Working test suite with at least:
  - 10 unit tests
  - 6 API tests
  - 3 Playwright smoke tests
- Confirm a deployment fails if a smoke test is forced to fail.
