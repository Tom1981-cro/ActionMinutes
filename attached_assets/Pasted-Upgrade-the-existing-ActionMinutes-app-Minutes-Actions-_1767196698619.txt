Upgrade the existing “ActionMinutes” app (Minutes → Actions → Follow-ups) to Phase 2 with: Calendar export (.ics), Gmail/Outlook draft creation via OAuth, Team spaces (multi-tenant), AI audit log, and role-based access control. Keep the app fast, minimal, and reliable. Do not bloat the MVP flows.

PHASE 2 GOALS
1) Calendar export (.ics) for meetings and optionally for action item due dates
2) Gmail + Outlook integrations to create drafts in the user’s mailbox (no auto-send)
3) Team spaces so multiple users can share meetings/action items with permissions
4) AI audit log to track model/prompt versions + outputs
5) RBAC enforcement across API + UI

NON-GOALS
- No automatic emailing or sending follow-ups
- No deep calendar two-way sync; export only
- No enterprise SSO yet (scaffold only)
- No complex billing yet (scaffold only)

A) CALENDAR EXPORT (.ics)
FEATURES
- From a meeting screen and from Extraction Review, add button: “Export to Calendar (.ics)”
- Export includes: meeting title, date, time/duration if present, location, and a description block that contains Summary + Decisions + Action items (as plain text)
- Add option toggle: “Include action items as separate calendar all-day tasks” (only if dueDate exists; otherwise skip)
- Provide “Download .ics” immediately; also store an export record in DB
SCREENS
- Meeting detail / review: Export button + modal to choose export options
- Settings > Exports: list of recent exports with download again
DATA
- Add CalendarExport model: id, userId, meetingId optional, createdAt, options JSON, filename, contentHash (store content or regenerate deterministically)

B) EMAIL DRAFT INTEGRATIONS (GMAIL + OUTLOOK)
FEATURES
- Settings > Integrations screen:
  - Connect Gmail (Google OAuth) with least scope needed to create drafts
  - Connect Outlook (Microsoft OAuth / Graph) with least scope needed to create drafts
  - Show status: Connected as {email}, Last used date, Disconnect button
- Draft Editor adds button:
  - “Create as Gmail Draft” (if Gmail connected)
  - “Create as Outlook Draft” (if Outlook connected)
- After creation: show “Draft created” + link if available OR confirmation message
SECURITY
- Store OAuth tokens securely (encrypted at rest if possible; otherwise use environment-based encryption key)
- Support revoke/disconnect: delete tokens
- Never send; only create drafts
DATA
- Add OAuthConnection model:
  - provider enum (google, microsoft)
  - userId
  - accountEmail
  - accessToken (encrypted)
  - refreshToken (encrypted)
  - expiry
  - scopes
  - createdAt, updatedAt, lastUsedAt

C) TEAM SPACES (MULTI-TENANT)
FEATURES
- Users can create a Workspace (Team)
- Invite members by email (simple invite token flow)
- Roles: Owner, Admin, Member, Viewer
- Meetings can belong to a workspace (optional); personal meetings still exist
- Action items can be assigned to a workspace member (ownerUserId)
- Inbox can filter: “My items” vs “Workspace items”
SCREENS
- Workspace switcher in header (Personal / Workspace)
- Workspaces screen:
  - Create workspace
  - Invite member
  - Members list + role management (Owner/Admin only)
- Meeting create/edit: choose workspace (optional)
RBAC RULES (ENFORCE SERVER-SIDE)
- Viewer: read-only
- Member: create/edit own meetings in workspace; edit action items if they own them or created them
- Admin: manage members (except owner), edit all workspace content
- Owner: full control, delete workspace
DATA
- Add Workspace model: id, name, createdByUserId, createdAt
- Add Membership model: id, workspaceId, userId, role enum, createdAt
- Add Invite model: id, workspaceId, email, token, expiresAt, createdAt, acceptedAt

D) AI AUDIT LOG
FEATURES
- Record each AI run:
  - timestamp
  - model/provider
  - prompt version
  - input hash (not the raw input unless user enables “store inputs”)
  - output JSON stored
  - validation status
- Admin/Owner can view audit log for their workspace; personal users can view their own audit log
SCREENS
- Settings > AI Audit:
  - list entries, filter by meeting, provider, date
  - open detail view showing outputs + validation errors
DATA
- Add AiAudit model: id, userId, workspaceId optional, meetingId optional, provider, model, promptVersion, inputHash, outputJson, validJson boolean, errorText optional, createdAt
